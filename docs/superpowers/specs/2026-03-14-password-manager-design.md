# OnePass 密码管理 App 设计文档

**日期：** 2026-03-14
**平台：** Android + macOS
**技术栈：** Flutter (Dart)

---

## 1. 项目概述

一款个人使用的密码管理 app，支持 Android 手机和 Mac 电脑，界面风格参考 1Password。数据完全本地化，通过局域网在两端之间同步，不依赖任何云服务。

**目标用户：** 个人使用，单一用户
**核心价值：** 安全、私密、跨设备可用

---

## 2. 功能范围（基础版）

### 包含功能
- 密码条目管理：创建、查看、编辑、删除（软删除）
- 条目字段：自定义名称、用户名、密码、网址（可选）、备注（可选）
- 分类管理：自定义分类（名称、emoji 图标、颜色）
- 收藏标记
- 全文搜索
- 密码生成器（长度、大小写、数字、特殊符号可配置）
- 主密码解锁 + 生物识别（指纹 / Touch ID）
- 局域网同步（Mac 作主端，Android 作客户端）

### 不包含功能
- TOTP 两步验证
- 文件附件
- 密码健康检查
- 自动填充
- 多用户 / 共享密码库
- 云同步

### 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    共享 Flutter 代码                      │
│  UI 层 / 业务逻辑 / 加密层 / 数据模型                      │
└──────────────┬──────────────────────┬───────────────────┘
               │                      │
    ┌──────────▼──────────┐  ┌────────▼────────────┐
    │    macOS App         │  │   Android App        │
    │  ┌───────────────┐  │  │  ┌───────────────┐  │
    │  │ SQLite 加密库  │  │  │  │ SQLite 加密库  │  │
    │  │ (SQLCipher)   │  │  │  │ (SQLCipher)   │  │
    │  └───────────────┘  │  │  └───────────────┘  │
    │  ┌───────────────┐  │  │  ┌───────────────┐  │
    │  │ HTTP 同步服务  │◄─┼──┼──│ 同步客户端     │  │
    │  │ (shelf 包)    │  │  │  │ mDNS 发现     │  │
    │  └───────────────┘  │  │  └───────────────┘  │
    └─────────────────────┘  └─────────────────────┘
              局域网（同一 WiFi）
```

### 核心组件

| 组件 | 职责 | 技术 |
|------|------|------|
| Vault 层 | 加密数据库读写 | SQLCipher (flutter_sqlcipher) |
| Crypto 层 | 密钥派生与数据加解密 | PBKDF2 + AES-256-GCM |
| Sync Server | 局域网 HTTP 服务（Mac 专属） | shelf 包 |
| Sync Client | 发现 Mac 并同步数据（Android 专属） | mDNS + HTTP |
| Auth 层 | 主密码验证 + 生物识别 | local_auth 包 |
| UI 层 | 跨平台界面 | Flutter Widgets |

## 4. 数据模型

### Entry（密码条目）

```dart
class Entry {
  String id;           // UUID
  String title;        // 用户自定义名称，如 "Google 工作"
  String username;     // 用户名 / 邮箱
  String password;     // 加密存储
  String? url;         // 网址（可选）
  String? notes;       // 备注（可选）
  String categoryId;   // 所属分类 ID
  bool isFavorite;     // 是否收藏
  DateTime createdAt;
  DateTime updatedAt;  // 用于增量同步判断
  DateTime? deletedAt; // 软删除时间
}
```

### Category（分类）

```dart
class Category {
  String id;     // UUID
  String name;   // 分类名称
  String icon;   // emoji 图标
  String color;  // 颜色十六进制值
}
```

**关于多账号处理：** 同一服务的多个账号通过自定义 `title` 区分（如"Google 工作"、"Google 个人"），不做自动分组，与 1Password 保持一致。

## 5. 安全模型

### 密钥派生

```
主密码（用户输入）
    │
    ▼
PBKDF2（100,000 次迭代，随机 salt，SHA-256）
    │
    ▼
256-bit 加密密钥
    │
    ├──► SQLCipher 数据库加密密钥
    └──► 存入系统安全区域
           ├── Android: Android Keystore（硬件级保护）
           └── macOS:   Keychain
```

### 生物识别解锁流程

1. 用户触发生物识别（指纹 / Touch ID）
2. 系统验证通过后，从 Keystore/Keychain 取出加密密钥
3. 用密钥解锁 SQLCipher 数据库
4. 主密码不存储，忘记主密码即无法恢复数据（设计如此）

### 局域网同步安全

- 同步请求携带 HMAC 签名（由主密码派生的 token 生成）
- Mac HTTP 服务只监听局域网 IP，不对外暴露
- 数据库层已加密，传输层为可信局域网

## 6. 同步协议

### 设备发现

Mac 通过 mDNS 广播自身（服务名：`_onepass._tcp`），Android 监听并自动发现 Mac 的 IP 和端口。

### 增量同步流程

```
Android                              Mac (HTTP Server)
  │                                        │
  │── GET /sync?since=<timestamp> ────────►│
  │                                        │ 返回所有 updated_at > since 的条目
  │◄── { entries: [...], serverTs: now } ──│
  │                                        │
  │── POST /sync { entries: [...] } ──────►│ 推送 Android 端的本地变更
  │◄── { ok: true } ──────────────────────│
```

### 冲突解决

- 策略：Last-Write-Wins（以 `updatedAt` 较新的记录为准）
- 软删除条目（`deletedAt` 非空）参与同步，确保删除操作跨端生效

```
┌──────────────┬───────────────────┬──────────────────────────┐

│  ├ 收藏       │  Google 工作      │                          │
│  └ 已删除     │  work@gmail.com   │  用户名  [复制]           │
│              │  Google 个人       │  密码    [显示] [复制]    │
```

### Android — 单栏布局

- **列表页**：顶部搜索栏 + 条目列表 + 底部导航栏（密码 / 分类 / 设置）
- **详情页**：返回按钮 + 条目信息 + 复制按钮
- **新建/编辑页**：表单 + 密码生成器入口

### 视觉规范

- 主题：深色（Dark）为主，与 1Password 风格一致
- 主色调：蓝色（`#0A84FF`）
- 背景：`#1C1C1E`（主）、`#2C2C2E`（次）、`#3A3A3C`（边框/输入）
- 字体：系统字体（iOS/macOS: SF Pro，Android: Roboto）

## 8. 页面结构

```
解锁页（启动入口）
    │ 验证成功
    ▼
密码列表（主界面）
    ├── 点击条目 ──► 条目详情 ──► 编辑条目 ──► 密码生成器
    ├── 点击 + ──────────────► 新建条目 ──► 密码生成器
    └── 点击设置 ─────────────► 设置页
                                  ├── 修改主密码
                                  ├── 管理分类
                                  ├── 同步状态 / 手动同步
                                  └── 导入 / 导出
```

### 密码生成器参数

- 长度：8–64 位（默认 16）
- 选项：大写字母、小写字母、数字、特殊符号（各自可开关）

## 10. 测试策略

- **单元测试**：Crypto 层（密钥派生、加解密正确性）、同步冲突解决逻辑、密码生成器
- **集成测试**：Vault 层 CRUD 操作、同步协议完整流程
- **Widget 测试**：核心 UI 组件（列表、详情、表单）
- **平台测试**：在真实 Android 设备和 macOS 上验证生物识别和局域网同步
